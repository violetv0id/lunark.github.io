Features (currently surviving)

Custom Rendering

Framebuffer-driven rendering, materials, lighting experiments, and constant shader tinkering.

Bepu Physics

Fast deterministic simulation wired directly into engine entities instead of hiding behind magic.

Entity System

Flexible component architecture designed for experimentation instead of ceremony.

Tooling

Debug views, runtime inspection, and tools built mostly because debugging without them was suffering.

Community Chaos

Ideas, feedback, and occasional cursed suggestions slowly shaping direction.

Early Development

This engine is unfinished. Parts are experimental. Some systems exist purely because curiosity won an argument.

The goal isn't polish yet — it's understanding. Build things, break them, rebuild them better.

Why Limeko Exists

Making engines teaches you what game development actually feels like under the hood. Limeko is a long-term experiment in control, clarity, and learning by doing things the hard way first.
